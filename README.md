# FinRelief AI вЂ” AI Powered Debt Relief & Financial Recovery Platform

A full-stack implementation of the FinRelief AI platform: borrowers register,
add loan accounts, get an automatic financial health assessment, receive
rule-based settlement predictions, and generate AI-powered negotiation
strategies and settlement letters (Google Gemini, with an automatic
rule-based fallback so the app always works end to end).

**Stack:** React.js (Vite) В· FastAPI В· Python В· SQLite В· SQLAlchemy ORM В· Google Gemini API

> **Python version:** written and tested against Python 3.12, and uses no
> syntax or library features that Python 3.14.2 would break вЂ” the code is
> forward-compatible. If your machine has Python 3.14.2 installed, `python3 -m venv venv`
> below will create the virtual environment with that interpreter automatically.

## 1. Backend setup (FastAPI)

```bash
cd backend
python3 -m venv venv

# macOS/Linux
source venv/bin/activate
# Windows
venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env
# then edit .env if you have a Gemini API key (optional вЂ” see below)

uvicorn app.main:app --reload --port 8000
```

The API is now live at `http://127.0.0.1:8000`, with interactive docs at
`http://127.0.0.1:8000/docs`.

### Gemini API key (optional)

Get a free key from https://aistudio.google.com/app/apikey and put it in
`backend/.env` as `GEMINI_API_KEY=...`. If you leave it blank, negotiation
strategies and letters are generated by a template-based fallback instead вЂ”
the platform works fully either way, which is exactly the "AI fallback
management" called for in Epic 5.

## 2. Frontend setup (React + Vite)

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open `http://127.0.0.1:5173`. The Vite dev server proxies any request to
`/api/*` through to the FastAPI backend on port 8000 (see
`vite.config.js`), so no extra CORS configuration is needed locally.

To build a production bundle:

```bash
npm run build   # outputs to frontend/dist
npm run preview # serve the production build locally
```

## 3. Using the app

1. **Register** a borrower account with a name, email, password, and
   starting monthly income/expenses.
2. **Add a loan** on the Loans page (lender, type, outstanding amount, EMI,
   interest rate, months overdue). This automatically:
   - Recalculates your financial profile (EMI ratio, DTI ratio, monthly
     surplus, stress level, financial health score).
   - Generates a settlement prediction (likely/possible/unlikely, a
     recommended settlement amount, risk category, and priority level).
3. Click **Negotiate** on a loan to generate an AI negotiation strategy and
   a draft settlement letter (Gemini if configured, otherwise the built-in
   fallback generator).
4. The **Overview** dashboard, **Financial profile**, **Settlements**, and
   **AI negotiations** pages surface all of the above with the borrower's
   full history.

## 4. Database (Epic 3)

SQLite is used by default (`backend/finrelief.db`, created automatically on
first run) via SQLAlchemy ORM, matching the ER diagram:

- `users` (1) в”Ђв”Ђ (1) `financial_profiles`
- `users` (1) в”Ђв”Ђ (N) `loans`
- `users` (1) в”Ђв”Ђ (N) `ai_history`
- `loans` (1) в”Ђв”Ђ (N) `settlement_predictions`
- `loans` (1) в”Ђв”Ђ (N) `ai_negotiations`

To move to PostgreSQL/MySQL later, just change `DATABASE_URL` in `.env` вЂ”
no other code changes are required since all access goes through
SQLAlchemy's ORM layer.

## 5. Security notes

- Passwords are hashed with bcrypt (never stored in plaintext).
- Authentication uses JWT bearer tokens (`python-jose`), issued on login and
  required on every protected endpoint via a FastAPI dependency.
- CORS is wide open (`allow_origins=["*"]`) for local development вЂ” restrict
  this to your deployed frontend's origin in `backend/app/main.py` before
  shipping to production.
- Change `SECRET_KEY` in `.env` to a long random value before any real
  deployment.
